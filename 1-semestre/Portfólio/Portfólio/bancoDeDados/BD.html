Banco de Dados

Um Banco de dados é um conjunto organizado de diversos dados, servindo como uma ferramenta para armazenar, gerenciar e acessar informações de maneira eficiente e segura.

Estrutura de Dados

Estrutura de dados é uma maneira especifica de organizar e armazenar os dados para que sua manipulação possa ser usada de forma mais agil e eficiente.

Diagrama de Relacionamento

Um diagrama de relacionamento é uma representação visual das conexões entre as tabelas em um banco de dados, mostrando como elas se relacionam entre si através de chaves primárias e estrangeiras.
Ele serve para visualizar e compreender a estrutura, facilitando a implementaçã e a manutenção do sistema.

PRIMARY KEY

Um valor único e obrigatório que identifica de forma exclusiva cada registro em uma tabela no banco de dados.

FOREIGN KEY

Um campo que refere-se a uma chave primária em outra tabela, estabelecendo um relacionamento entre as duas tabelas.

Comandos

Os comandos em um banco de dados são cruciais para a interação com os dados armazenados. Eles permitem criar, ler, atualizar e excluir dados, alem de administrar a estrutura do banco e controlar o acesso dos usuários. Abaixo mostrarei exemplos dos comandos mais usados.

Comandos DDL

CREATE TABLE - Criação de tabela
ALTER TABLE - Adiciona, altera ou exclui tabelas
DROP TABLE - Comando para apagar a tabela

Comandos DML

INSERT - Usado para inserir dados na tabela
SELECT - Usado para visualizar a tabela

Comandos DQL

SELECT - Selecione um dado para visualização
JOIN - Serve para combinar tabelas

Existem diversas combinações de comandos para a visualização necessaria, no meu GitHub tenho diversos exemplos de exercicios usando operadores e parâmetros.

Normalização

A normalização é um processo de organização de tabelas em um banco de dados para reduzir redundâncias, evitar problemas de atualização e garantir a consistência dos dados, dividindo-os em formas normais. Isso ajuda a melhorar a eficiência e a integridade dos dados armazenados.

Bando de Dados NoSQL

Um banco de dados NoSQL é um sistema de banco de dados que não utiliza o modelo relacional. Ele é projetado para armazenar e processar dados não estruturados ou semiestruturados, oferecendo flexibilidade e escalabilidade horizontal, como por exemplo o MongoDB.

NoSQL oferece varias vantagens como permitir distribuir dados por múltiplos servidores, garantindo alta disponibilidade e expansão.
Permitem uma flexibilidade de esquema, um alto desempenho para operações rapidas de leitura e escrita, e uma facilidade em seu gerenciamento.

MongoDB

MongoDB é um banco de dados NoSQL orientado a documentos que armazena dados em formato JSON. Suas principais vantagens incluem escalabilidade horizontal e flexibilidade de esquema, ideal para aplicações que lidam com grandes volumes de dados variados.
